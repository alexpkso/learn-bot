# Инструкция для Claude Code: голосовое принятие заказа (мобильная главная)

Контекст: на **главной странице** (`app/page.tsx`) в **мобильной вёрстке** (`sm:hidden`) есть блок **«Быстрый заказ»** с кнопкой **микрофона** для голосового ввода. Речь превращается в текст, затем в поля формы заказа.

## Где в коде

| Что | Файл |
|-----|------|
| UI блока «Быстрый заказ» + микрофон | `app/page.tsx` — секция с комментарием `Мобильный блок: быстрый ввод заказа`, класс обёртки `sm:hidden` |
| Состояние | `voiceListening`, `voiceTranscript`, refs `recognitionRef`, `micStreamRef` |
| Запись речи | `startVoiceInput()`, `stopVoiceInput()`, `releaseMicrophone()` — Web Speech API (`SpeechRecognition` / `webkitSpeechRecognition`) |
| Разбор текста в поля | `parseVoiceInput(text)` → `POST /api/parse-voice` |
| Серверный разбор (Claude) | `app/api/parse-voice/route.ts` — Anthropic SDK, модель `claude-haiku-4-5` |

## Установка / окружение

1. **Переменная окружения** (Vercel / `.env.local`):
   - `ANTHROPIC_API_KEY` — ключ API Anthropic; без него `parse-voice` не сможет вызывать модель (клиент получит ошибку; в коде есть **fallback**: грубый локальный разбор числа и типа продукции в `catch` у `parseVoiceInput`).

2. **Зависимости**: пакет `@anthropic-ai/sdk` уже используется в `app/api/parse-voice/route.ts`.

3. **Браузер**: голос работает только там, где есть **Web Speech API** (часто Chrome / Edge на Android и десктоп; Safari может отличаться). Нужен **HTTPS** (или localhost) для `getUserMedia` и нормальной работы распознавания.

4. **Микрофон**: перед распознаванием запрашивается `navigator.mediaDevices.getUserMedia({ audio: true })` — пользователь должен выдать разрешение.

## Поведение (кратко)

- Язык распознавания: **ru-RU**.
- Режим **continuous**: запись идёт, пока пользователь не нажмёт **«Готово»** (останавливает распознавание и поток микрофона).
- После остановки (`onend`) вызывается `parseVoiceInput(fullTranscript)` с полным текстом.
- API возвращает JSON с полями вроде `product_type_id`, `quantity`, `client_id`, `new_client_name`, `phone`, `contact_person` — они подставляются в `quickForm` (и при необходимости включаются режимы нового клиента / нового типа продукции).

## Что не путать

- Блок **только для мобильной ширины** (`sm:hidden`). На `sm+` этот быстрый заказ с микрофоном **не показывается** — менять десктоп ищите отдельно.
- Голос **не** заменяет ручное заполнение: после разбора пользователь может править форму и нажать отправку как обычно.

## Типичные доработки (для агента)

- Изменить подсказки / поля в промпте — править строку `prompt` в `app/api/parse-voice/route.ts`.
- Поменять модель — поле `model` в `client.messages.create`.
- UI микрофона / тексты «Слушаю…», «Готово» — только в `app/page.tsx` внутри `sm:hidden` секции.
